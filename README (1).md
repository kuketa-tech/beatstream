# BeatStream

**BeatStream** — это клиент‑серверное музыкальное приложение на Python с desktop‑интерфейсом на **PyQt6** и собственным TCP/JSON‑сервером для хранения пользователей, треков и плейлистов.

Проект позволяет:
- регистрироваться и входить в аккаунт;
- загружать MP3‑треки на сервер;
- просматривать общую библиотеку;
- искать музыку по названию, исполнителю, альбому и жанру;
- ставить лайки трекам;
- создавать и редактировать плейлисты;
- воспроизводить музыку в desktop‑клиенте.

---

## Возможности

### Клиент (PyQt6)
- современный desktop UI;
- авторизация и регистрация;
- страница библиотеки с фильтрацией по жанру и сортировкой;
- поиск по библиотеке;
- раздел «Понравилось»;
- управление плейлистами;
- встроенный аудиоплеер с прогрессом воспроизведения и регулировкой громкости.

### Сервер
- TCP‑сервер на Python;
- обмен сообщениями в формате JSON;
- хранение пользователей, библиотеки и плейлистов в JSON‑файлах;
- хранение загруженных MP3 в папке `music_files/`;
- учёт лайков, прослушиваний и даты загрузки;
- рекомендации на основе популярности треков;
- базовая статистика по системе.

### Скрипт загрузки
В проекте есть отдельный скрипт `upload.py`, который позволяет отправлять MP3‑файлы на сервер без использования GUI.

## Стек технологий

- **Python 3.10+**
- **PyQt6** — графический интерфейс
- **socket** — TCP‑взаимодействие между клиентом и сервером
- **JSON** — хранение данных и формат обмена
- **QMediaPlayer / QAudioOutput** — воспроизведение аудио

## Структура проекта

Ниже — рекомендуемая структура репозитория:

```text
BeatStream/
├── client.py              # desktop-клиент на PyQt6
├── server.py              # TCP/JSON-сервер
├── upload.py              # загрузка MP3 на сервер
├── users.json             # пользователи
├── music_library.json     # библиотека треков
├── playlists.json         # плейлисты
├── music_files/           # загруженные MP3-файлы
└── README.md
```

> Если у тебя сейчас серверный и клиентский код лежат в файлах с одинаковым именем `main.py`, для GitHub лучше переименовать их в `server.py` и `client.py`.

## Запуск проекта

### Запуск сервера

```bash
python server.py
```

После запуска сервер начнёт слушать подключения на:

```text
0.0.0.0:5555
```

### Запуск клиента

```bash
python client.py
```

Клиент по умолчанию подключается к:

```text
localhost:5555
```

## Загрузка музыки

Можно загрузить трек через вспомогательный скрипт:

```bash
python upload.py
```

Пример использования функции внутри `upload.py`:

```python
upload('song3.mp3', 'Лето', 'Вивальди', genre='Классика')
```

Скрипт:
- читает MP3-файл;
- кодирует его в Base64;
- отправляет на сервер тип сообщения `upload_song`;
- получает ответ с метаданными загруженного трека.

## Как это работает

### Авторизация
Сервер поддерживает:
- `register`
- `login`

Пароли хранятся в виде **SHA‑256 hash**.

### Работа с музыкой
Сервер поддерживает операции:
- загрузка трека;
- получение библиотеки;
- поиск;
- получение файла трека;
- лайк / снятие лайка;
- удаление трека;
- получение списка жанров;
- рекомендации.

### Работа с плейлистами
Поддерживаются команды:
- создание плейлиста;
- получение плейлистов пользователя;
- добавление трека в плейлист;
- удаление трека из плейлиста;
- удаление плейлиста.

## Формат хранения данных

### `users.json`
Хранит:
- username
- hashed password
- email
- liked songs
- playlists
- даты регистрации и последнего входа

### `music_library.json`
Хранит:
- id трека
- title
- artist
- album
- genre
- duration
- uploaded_by
- upload_date
- likes
- plays
- filename
- file_size

### `playlists.json`
Хранит:
- id плейлиста
- имя
- владельца
- описание
- список треков
- дату создания

## Ограничения

- сервер принимает один JSON‑запрос на соединение;
- максимальный размер входящего JSON‑запроса — **30 MB**;
- клиент по умолчанию подключается только к `localhost:5555`;
- проект использует JSON‑файлы вместо полноценной БД, поэтому лучше подходит для учебных и pet‑проектов.
